# 🏢 EconomyEX — Metropolis Economy Fix and Performance Boost

**For**: Cities: Skylines 2 · **Map Size**: Standard / Vanilla (14 km) only

## ⚠️ Compatibility (Read First)

This mod replaces multiple vanilla economic simulation systems. **It is likely to conflict with any other mod that also modifies economy, job search, or pathfinding-related systems.**

Known conflicts:
- **MapExt (MapExtPDX)**: Do NOT install both. MapExt already includes this economy module built-in(and usually updates faster). If both are subscribed, EconomyEX will auto-disable itself.
- **Realistic PathFinding / Realistic JobSearch / Realistic Trips**: Confirmed or reported conflicts. If you must use them together, disable the conflicting system groups individually in EconomyEX's settings panel.
- **MapExt (28 km+)**: This mod auto-detects map size and only activates on vanilla maps.
- **Asset / Visual Mods**: No conflict expected.

---

## 📋 Overview

This mod is a standalone version of the economic patch from the MapExt (bigger map size) mod, applicable only to the vanilla 14 km map.

Several core economic systems in the vanilla game are only tuned for cities with smaller populations (200k–500k). When a city's population reaches the million-level, the following issues may occur:
1. Imbalanced demand, housing vacancies, and job mismatches;
2. Systems such as resident housing searches, consumption behavior, and service coverage consuming excessive performance;
3. Land value and rent calculations producing unreasonable results.

This mod modifies these vanilla core economic systems to make them more suitable for simulating million-plus metropolis cities, and reduces unnecessary calculations while preserving the original logic as much as possible. For specific code modifications, please refer to GitHub.

## 🔧 What This Mod Changes

### 📈 Demand Calculation
- The vanilla RCI demand calculation uses absolute values, which cause demand to permanently lock at zero in large cities. This mod converts them to percentage-based, so demand remains responsive regardless of population size.
- Commercial demand now scales linearly with consumer count; industrial labor metrics are normalized.

### ⚡ Job Search and Shopping Trips
- Reduces the frequency of job search and shopping trip pathfinding at high population, to lower CPU load.
- Limits the number of simultaneous homeless pathfinding requests per frame to prevent sudden lag spikes.

### 🗺️ Land Value
- In vanilla, the game calculates two separate land values: one used in economic simulation (Edge-based) and another only displayed on the UI heatmap. These two values use different calculation methods, so the land value infoview does not reflect the actual value used in the economy.
- This mod unifies the two: the Edge land value is calculated with all factors (service coverage, environment, pollution), and the heatmap is generated by sampling the Edge values. This makes the UI match what the economy actually uses.

### 🏘️ Rent
- In vanilla, household eviction decisions are overly aggressive. This mod restores the calculation of `Income + Savings` as the baseline for determining whether a household can afford rent, preventing mass eviction cascades.
- Households with very low rent-to-income ratios will only occasionally seek better housing, rather than all moving at once. This reduces unnecessary pathfinding load.

### 🕹️ Resident AI
- Fixes a logic defect in resident pathfinding wait-time that could cause excessive memory usage at large population scales.

## 🎚️ Pathfinding Cost Sliders (In-Game)

In the mod's settings panel (Options -> EconomyEX), you can adjust five pathfinding cost limits in real time. These values cap how far citizens and companies are willing to search. **Changes take effect immediately, no restart required.**

| Slider | What it controls | Range | Default |
|--------|-----------------|-------|---------|
| Shopping Max Cost | Maximum travel cost a citizen will pay to go shopping | 1,000 – 200,000 | 8,000 |
| Company Shopping Max Cost | Maximum search range for companies restocking goods | 1,000 – 200,000 | 200,000 |
| Leisure Max Cost | Maximum travel cost for leisure trips (parks, landmarks) | 1,000 – 200,000 | 12,000 |
| Find Job Max Cost | Maximum search range for job seeking | 17,000 – 200,000 | 200,000 |
| Find Home Max Cost | Maximum search range for finding a new home | 17,000 – 200,000 | 200,000 |

**Tip**: Lower values restrict citizens to nearby options, which reduces pathfinding load but may cause mismatches. Higher values allow broader searches but consume more CPU. The defaults work well for most cities.

# 🏢 EconomyEX — 大型都市经济修复与性能优化

**适用**: Cities: Skylines 2 · **地图**：仅标准/原版地图（14 km）

## ⚠️ 兼容性（请先阅读）

本模组替换了多个原版经济模拟系统。**凡是涉及经济、求职、寻路等系统修改的模组，都可能与本模组冲突。**

已知冲突：
- **MapExt (MapExtPDX)**：请勿同时安装。MapExt 已经内置了完全相同的经济模块(一般会比这个独立版本優先更新)。如果同时订阅，EconomyEX 会自动休眠。
- **Realistic PathFinding / Realistic JobSearch / Realistic Trips**：已确认或有用户报告存在冲突。如需同时使用，请在 EconomyEX 设置面板中将冲突的系统组单独关闭。
- **大地图（28 km 以上）**：本模组会自动检测地图大小，仅在原版尺寸地图上生效。
- **资产/视觉类 Mod**：无冲突。

---

## 📋 简介

本模组是 MapExt（大地图）中经济补丁的独立版本，仅适用于原版 14 km 地图。

原版游戏若干核心经济系统的算法和参数仅适合中小城市（20万–50万人口）。当城市人口达到百万级时，可能出现以下问题：
1. 需求失衡、住房空置与岗位错配；
2. 居民找房、消费出行、服务覆盖等系统消耗过多性能；
3. 地价和租金计算产生不合理的结果。

本模组修改了这些原版核心经济系统，使其更适用于百万级大都市的模拟，并在尽量保留原版逻辑的前提下减少不必要的计算。具体代码修改请参阅 GitHub。

## 🔧 具体修改内容

### 📈 需求计算
- 原版 RCI 需求计算使用绝对数值，导致大城市需求条永远为零。本模组将其改为百分比制，使需求响应不受人口规模影响。
- 商业需求随消费者数量线性增长；工业劳动力指标做了标准化处理。

### ⚡ 求职与购物出行
- 在高人口下降低求职和购物出行的寻路频率，减轻 CPU 负担。
- 限制每帧同时进行的无房家庭寻路请求数量，防止瞬间卡顿。

### 🗺️ 地价
- 原版游戏中存在两套地价：一套用于经济模拟计算（道路边 Edge 地价），另一套仅用于 UI 热力图显示。两者计算方式不同，导致地价信息图与实际经济计算所用的值不一致。
- 本模组将两者统一：在 Edge 地价中直接计入所有因素（服务覆盖、环境、污染等），热力图通过采样 Edge 地价生成。使 UI 显示与经济模拟使用同一套数据。

### 🏘️ 租金
- 原版的家庭驱逐判定过于激进。本模组恢复以"收入 + 存款"作为家庭能否负担租金的判定基准，防止因存款不合理导致大规模连锁驱逐。
- 租金占收入比极低的家庭仅有一定概率寻求改善住房，而非全部同时搬迁，减少不必要的寻路开销。

### 🕹️ 居民 AI
- 修复居民寻路等待时间的逻辑缺陷，降低大规模人口下内存占用过高的风险。

## 🎚️ 寻路成本滑块调节（游戏内）

在模组设置面板（选项 -> EconomyEX）中，可以实时调节以下五个寻路成本上限。这些值决定了市民和公司愿意搜索的最大范围。**修改后立即生效，无需重启。**

| 滑块 | 含义 | 范围 | 默认值 |
|------|------|------|--------|
| 购物最大成本 | 市民购物出行的最大寻路成本 | 1,000 – 200,000 | 8,000 |
| 公司采购成本 | 工厂/商店补货的最大搜索范围 | 1,000 – 200,000 | 200,000 |
| 休闲最大成本 | 休闲出行（公园、地标等）的最大成本 | 1,000 – 200,000 | 12,000 |
| 找工作成本 | 求职搜索的最大范围 | 17,000 – 200,000 | 200,000 |
| 找房成本 | 搬家找房的最大搜索范围 | 17,000 – 200,000 | 200,000 |

**提示**：值越小，市民活动范围越局限（寻路压力小，但可能出现匹配不上的情况）；值越大，搜索范围越广（模拟更真实，但 CPU 开销更大）。默认值适用于大多数城市。
